### 2.2 需新增后端端点

1. **`POST /api/v1/ai/dify-proxy`**（薄代理）
   - body：`{ app: 'selection'|'align'|'insight'|'questions'|'virtual_shelf', path: 'workflows/run'|'files/upload', payload }`
   - 后端从 env 取对应 `DIFY_KEY_*`，转发到 `DIFY_BASE_URL/${path}`，**stream 模式直接 pipe**（原 service 依赖 NDJSON 流读）
   - **AI key 留后端，前端永远不接触** ← 这是用户硬约束
   - 已有 `dify.service.ts` 是结构化调用；这里专门做"流式代理"补丁

2. **`POST /api/v1/detect`** 改造（已存在但接口形态不对）
   - 现在：JSON `{imageBase64}` → 返回 `{boxes:[{x,y,w,h,skuCode,confidence}]}`
   - 原 repo 需要：multipart `image` → 返回 `matches:[{id, bbox:[x1,y1,x2,y2], matched_sku_id, matched_combined}]`
   - 改造方案：路由层加 `multer.single('image')`，base64 与 multipart 两条路径都支持；返回体加 `matches` 别名（保留 `boxes` 不破坏现有调用）

3. **`GET /api/v1/master/store-skus`**（如果未有）
   - 选品页 `getStoreSkuData(storeId)` 依赖该数据
   - 走 session.currentStoreId，返回 `[{ skuCode, skuName, majorCategory, midCategory, subCategory, sales30d, salesVolume30d, salesChange30d, shelfLifeDays, spec }]`
   - 若 `master.routes.ts` 已有等效接口，shim 内做映射；否则加端点

4. **`GET /api/v1/master/promo-skus`**（小，可选）
   - 取 promo_groups 表，返回 sku 集合，用于"活动"徽标

### 2.3 Detect 服务

- 沿用 `/tmp/skuSelection/detect-service/`，需要 GPU/PE 模型/Qdrant，**集成测试阶段不强求跑起来**
- `detect.service.ts` 已有 `DETECT_SERVICE_URL` env；不可达时改为 502，shim 内 catch → 返回 `[]`（前端表现：没有红框，但诊断/选品仍能继续）
- 把 detect-service 目录复制一份到 `services/detect/` 留作部署素材（不进 web/api 构建）

## 5. 改造细节

### 5.1 React Router → TanStack Router

| 原 | 新 |
|---|---|
| `useNavigate()` | `useNavigate({ from: Route.fullPath })` |
| `navigate('/position/0/photo')` | `navigate({ to: '/shelves/position/$code/photo', params: { code: '0' } })` |
| `useParams<{code: string}>()` | `Route.useParams()` |

> 改造原则：把这两个引用集中到一个 wrapper hook（如 `useSceneNav()`），各页面只用 wrapper，下次再改路由系统也不会乱炸。

### 5.2 selectedStore 注入

原 `AppContext` 里 `selectedStore` 是用户登录后从 store 列表里手动选的字符串编号（如 `粤37893`）。整合 app 里 `useMe()` 的 `currentStore.code` 就是这个值。

新 shim：

```tsx
// apps/web/src/components/shelves/AppShell.tsx
export function ShelvesAppShell({ children }: { children: ReactNode }) {
  const { data: me } = useMe();
  const storeCode = me?.currentStore?.code ?? '';
  return (
    <AppContext.Provider value={{ selectedStore: storeCode, ...originalDefaults }}>
      {children}
    </AppContext.Provider>
  );
}
```

每个路由的根 component 包一层 `ShelvesAppShell` 即可。

### 5.3 Dify 流式代理

原 `lib/difyWorkflowStream.ts` 直接 fetch 拼了 `/api/dify-proxy?app=X&path=Y` 流式读。后端新增：

```ts
// apps/api/src/routes/ai.routes.ts 内追加
aiRouter.post('/ai/dify-proxy', requireAuth, asyncHandler(async (req, res) => {
  const { app, path } = req.query;  // 'selection' / 'workflows/run'
  const key = pickDifyKey(app);     // 从 env.DIFY_KEY_* 取
  const upstream = await fetch(`${config.DIFY_BASE_URL}/${path}`, {
    method: 'POST',
    headers: { Authorization: `Bearer ${key}`, 'Content-Type': 'application/json' },
    body: req.body,  // 透传
  });
  res.status(upstream.status);
  upstream.headers.forEach((v, k) => res.setHeader(k, v));
  upstream.body!.pipeTo(Writable.toWeb(res));
}));
```

前端 shim：
```ts
// lib/sku-selection/dify-proxy.ts
export const difyProxyUrl = (app: string, path: string) =>
  `/api/v1/ai/dify-proxy?app=${app}&path=${encodeURIComponent(path)}`;
```

原 `src/lib/difyProxyUrl.ts` 用这个替换。

### 5.4 Detect 改造

后端：

```ts
// apps/api/src/routes/detect.routes.ts
import multer from 'multer';
const upload = multer({ storage: multer.memoryStorage(), limits: { fileSize: 12 * 1024 * 1024 } });

detectRouter.post('/detect', requireAuth, upload.single('image'),
  asyncHandler(async (req, res) => {
    const buf = req.file?.buffer ?? Buffer.from(req.body?.imageBase64 ?? '', 'base64');
    if (!buf.length) throw new AppError(400, ErrorCodes.BAD_REQUEST, '图片为空');
    try {
      const r = await detectService.detect(buf, req.file?.originalname ?? 'shelf.jpg');
      // 兼容原 repo 形态
      res.json(r.boxes.map((b, i) => ({
        id: String(i),
        bbox: [b.x, b.y, b.x + b.w, b.y + b.h],
        matched_sku_id: b.skuCode || null,
        matched_combined: b.confidence,
      })));
    } catch (err) {
      if ((err as AppError).status === 502) return res.json([]);  // 不阻塞业务流
      throw err;
    }
  }));
```

// lib/sku-selection/storage.functions.ts
export async function uploadPhoto(blob: Blob, _storeId: string, shelfId: string): Promise<string> {
  const fd = new FormData();
  fd.append('file', blob, `shelf-${Date.now()}.jpg`);

## 7. 风险 / 已知限制

1. **Python 检测服务依赖 GPU + PE 模型 + Qdrant**
   - 不在本次 PR 范围内拉起；前端容错 → 流程可走通，仅缺红框标注
   - 部署素材保留在 `services/detect/`，后续单独工单跑起来

2. **Dify keys 必须配置 5 个**
   - `DIFY_KEY_SELECTION / ALIGN / INSIGHT / QUESTIONS / VIRTUAL_SHELF`
   - 若缺失：诊断/选品/虚拟货架失败但不崩页（已有 try/catch + 错误提示）

3. **OSS 必须可用**
   - 上传走 `/storage/upload`；本地开发若 OSS_* 未配置，已有 fallback（写 /tmp 或返回 data URL，看 `oss.service.ts` 实现）

4. **跨 store 隔离**
   - 用户硬约束：业务接口不接受 `storeId` 入参，全部从 session 取
   - 所有 shim 调 v1 端点时不传 storeId，仅 path/body 必须的业务字段

5. **前端体积膨胀**
   - 整个选品模块约 +3k 行 TS/TSX；与 posters 量级相当。考虑路由 lazy import：
     ```tsx
     export const Route = createFileRoute('/shelves/position/$code/photo')({
       component: lazyComponent(() => import('@/components/shelves/PhotoPage')),
     });
     ```

6. **路由参数 `$code` 类型**
   - TanStack Router param 默认 string；要 `Number($code)` 并校验 `Number.isFinite`，否则跳回 `/shelves/position`
